# FigmaAI フロントエンド制作タスク

## 1. 目的
- Wi-Fi／URL QR コード生成アプリのフロントエンドデザイン（モックアップ＋UI キット）を Figma 上で作成する。
- バックエンドは Go で同一 Docker コンテナ内に配置されるが、フロント側は純クライアント処理（QR 生成、i18n）を前提とする。

## 2. 作業範囲
1. **単一ページ構成**
   - 上部トグルで Wi-Fi／URL モードを切り替える。
   - Wi-Fi モード: SSID／パスワード入力フィールド＋生成ボタン（もしくは自動生成）。
   - URL モード: URL 入力フィールドのみ。
   - QR プレビュー領域と PNG ダウンロードボタン。
   - 入力注意書き（例: 「SSID は必須」「URL には https:// を含めてください」）。
2. **i18n 対応**
   - 日本語／英語の UI テキストを想定し、文字量変化を吸収できるコンポーネント設計。
   - 文言は JSON キーを想定したラベルで記載（例: `label.ssid`, `action.download`）。
3. **デザイン指針**
   - 白／黒を基調にしたミニマルでフラットな画面。余計な装飾・情報は排除。
   - タイポグラフィはシステムフォント系列（例: `system-ui`, `Segoe UI`, `Noto Sans JP`）を想定。
   - 300px 以上の QR 表示枠を確保。余白を十分取り、視線誘導を明瞭にする。
4. **レスポンシブ**
   - 375px 幅（モバイル）と 1440px 幅（デスクトップ）の２ブレークポイントでレイアウト案を用意。

## 3. 成果物
- Figma ファイル内に以下を含める：
  - ページ構成図（Wi-Fi モード、URL モード）。
  - コンポーネントセット（トグル、入力、ボタン、注意書きテキスト、QR プレビュー枠）。
  - day/night ではなく白黒基調の単一テーマ。
- エクスポートガイド（PNG ダウンロードボタンなどのアセット指定）。SVG は不要。

## 4. 技術的背景／制約
- バックエンド：Go（静的配信・同一 Docker コンテナ内で稼働）。API 通信は行わず、静的ファイル提供のみ。
- フロント：SPA（JavaScript）で QR を生成し、`i18next` による i18n 切替を実装予定。
- 生成された QR コードはクライアントで `canvas`→PNG としてダウンロード。
- セキュリティ上、入力データはブラウザ外に送らない。

## 5. ガイドライン
1. **情報設計**
   - 画面内の情報は「モード切替」「フォーム」「QR プレビュー」「PNG ダウンロード」「注意書き」程度に限定。
   - ステータス／エラー表示はコンパクトにまとめる（フォーム直下）。
2. **アクセシビリティ**
   - コントラスト比 4.5:1 以上を確保。
   - トグルやボタンにはフォーカスリングを想定したスタイル表現を付与。
3. **構成管理**
   - レイヤー命名、コンポーネント命名に英語を使用し、開発連携しやすくする（例: `Toggle/WifiUrl`, `Button/Primary`）。
4. **Docker/Go 連携メモ**
   - フロントエンドは同一コンテナ内で Go サーバーから配信される想定のため、ビルド成果物が `public/` 等の静的ディレクトリに置かれる前提で整理する。
